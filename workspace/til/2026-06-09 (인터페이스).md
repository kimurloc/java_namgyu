#### 13.4 추상 클래스
실체 클래스들의 공통적인 특성(필드, 메서드)을 추출하여 선언한 미완성 설계도. 클래스 선언부에 `abstract` 키워드를 붙여 정의한다.
```java
abstract class Animal {}
```
- 직접적인 객체 생성 불가 : 설계도 자체가 미완성이므로 new 연산자를 사용하여 단독으로 **인스턴스를 생성할 수 없음**
- 생성자 보유 : 직접 객체를 생성할 수는 없지만, 자식 객체가 생성될 때 부모 생성자인 `super()`를 호출해야 하므로 생성자가 반드시 존재하고 호출됨
- 일반 멤버 포함 가능 : 미완성인 추상 메서드 외에도 일반적인 필드(멤버 변수)와 완성된 바디를 가진 일반 메서드를 자유롭게 가질 수 있음
실체가 없는 추상적인 개념의 클래스가 new 연산자를 통해 단독 객체로 만들어지는 실수를 방지하기 위해 사용한다.
#### 13.5 추상 메서드
선언부만 있고 구현부가 없는 메서드로, `abstract` 키워드를 사용하여 선언
```java
abstract class Animal { abstract void sound(); }
```
추상 클래스를 상속받은 자식 클래스는 부모의 모든 추상 메서드를 반드시 오버라이딩하여 구체적으로 구현해야 하며, 그렇지 않을 경우 자식 클래스 역시 추상 클래스로 선언해야 하므로 컴파일 에러가 발생한다. 즉 **추상 메서드**는 **자식 클래스에게 특정 동작의 구현을 강력하게 강제하기 위해 사용**한다.

#### 13.6 인터페이스
인터페이스는 객체의 사용 설명서이자 극단적인 추상화 설계도로 구현부 없이 오직 객체가 수행해야 할 동작의 규격(스펙)만을 정의해 둔 것이며, 클래스들이 가져야 할 기능의 공통 표준 규격을 마련하기 위해 사용한다.
- 클래스 선언 시 사용하는 `class` 키워드 대신 `interface` 키워드를 사용하여 선언
- 인터페이스 내의 멤버 변수는 컴파일 시 자동으로 `public static final` 상수로 취급되며, 반드시 선언과 동시에 초기화해야 함
- 인터페이스 내부의 메서드는 기본적으로 구현부(중괄호 블록)가 없는 추상 메서드로 선언됨 (자동으로 `public abstract` 제어자 지정)
- 자바는 클래스 상속의 단일 상속 제약을 극복하기 위해 인터페이스를 통한 다중 구현을 완벽히 지원함. 하나의 클래스가 여러 인터페이스를 `implements` 키워드 뒤에 콤마(,)로 나열하여 동시에 구현할 수 있음
- 추상 클래스와 마찬가지로 new 연산자를 사용하여 단독으로 인스턴스화할 수 없음
```java
interface Calculator {
	double PI = 3.14159;
	int calculate(int a, int b);
}
interface Printer { void printResult(int result); }

class MultiCalculator implements Calculator, Printer {
  public int calculate(int a, int b) { return a + b; }

  public void printResult(int result) {
    System.out.println("results : " + result);
  }
}
```
