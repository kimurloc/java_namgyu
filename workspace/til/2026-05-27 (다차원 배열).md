#### 9.4 다차원 배열
```java
type[][] arrayName = new type[rows][cols];
```
---
```java
int[][] arr = {{1, 2}, {3, 4}, {5, 6}};
int length = arr.length // 3
```
Q. 2차원 배열의 .length가 전체가 아닌 행의 크기인 이유?

A. 자바에서 2차원 배열은 실제로 **배열을 원소로 가지는 1차원 배열**이다. 즉 `arr`의 행은 `int[]`배열을 담고 있는 1차원 배열이므로, `arr.length`는 행의 크기가 된다. 또한, 열의 크기가 서로 다른 가변 배열에서도 행은 각 열 배열을 원소로 가지는 배열이므로 `.length`는 행의 크기 이다.

#### 9.5 향상된 for문
```java
int[] scores = {90, 85, 95};
for (int score : scores) {
    System.out.println(score + " ");
}
```
배열이나 `컬렉션`의 모든 요소를 처음부터 끝까지 순차적으로 탐색할 때 인덱스 없이 사용할 수 있고, 인덱스를 직접 제어하지 않으므로 인덱스 범위를 초과하는 에러를 예방할 수 있다. 단, 값을 조회하는 용도로만 사용이 가능하며 배열의 요소를 직접 수정할 수는 없다.
> `컬렉션`은 **다양한 종류의 데이터를 효율적으로 처리하기 위한 자료구조 클래스의 모음**이다.

#### 9.6 얕은 복사(Shallow Copy)
- 배열 변수의 주소 값만 복사하여 두 참조 변수가 동일한 실제 배열 객체를 가리키므로, 한쪽에서 값을 수정하면 다른 쪽도 값이 변경된다.
```java
int[] original = {10, 20, 30};
int[] shallow = original;

shallow[0] = 99;
System.out.println(shallow[0]); // 99
System.out.println(original[0]); // 99
```

#### 9.7 깊은 복사(Deep Copy)
- 새 배열을 생성한 뒤 기존 배열의 데이터를 복사하는 방식이므로, 한쪽에서 값을 수정해도 다른 쪽은 영향을 받지 않는다.
```java
int[] original = {10, 20, 30};
int[] target = new int[original.length];
for (int i = 0; i < original.length; i++) {
  target[i] = original[i];
}

target[0] = 99;
System.out.println(target[0]); // 99
System.out.println(original[0]); // 10
```
- **System.arraycopy()
	- JVM 레벨에서 메모리를 직접 복사하여 고속으로 배열 복사를 수행하는 메서드
	  ```java
	  System.arraycopy(Object src, int srcPos, Object dest, int destPos, int length);
	  ```
	- `src`, `dest` : 원본 / 대상 배열
	- `srcPos`, `destPos` : 원본 / 대상 배열에서 복사가 시작될 index
	- `length` : 복사할 요소의 개수
- **Arrays/copyOf()**
	- `java.util.Arrays` 클래스가 제공하는 헬퍼 메서드
	- 대상 배열을 미리 new 연산자로 선언할 필요 없이, 복사와 동시에 새 배열 객체를 한 줄로 얻고자 할 때 적합
	- `Arrays.copyOf()`는 `System.arraycopy()`를 내부에서 호출하는 `Wrapper` 클래스
	  >`Wrapper 클래스`는 **기본 자료형(Primitive Type)을 객체(Object)로 감싸는 클래스**이다.
	- 
	  ```java
		import java.util.Arrays;

		int[] original = {10, 20, 30};
		int[] target = Arrays.copyOf(original, original.length);

		target[0] = 99;
		System.out.println(target[0]); // 99
		System.out.println(original[0]); // 10
	  ```

#### 9.8 배열의 예외처리
- **NullPointerException**
	- 참조 변수가 null인 상태에서 인덱스를 지정할 때 발생
- **ArrayIndexOutOfBoundsException**
	- 배열 인덱스의 유효 범위를 벗어난 인덱스를 참조하려고 할 때 발생

---
```java
int[] arr = {{1, 2, 3}, {2, 1, 3}, {1, 2, 3}};
boolean isEqual1 = true;
boolean isEqual2 = true;
boolean isEqual3 = true;

for(int i = 0; i < arr[0].length; i++) {
	if(arr[0][i] != arr[1][i]) {
		isEqual1 = false;
	}else if(arr[1][i] != arr[2][i]) {
		isEqual2 = false;
	}else if(arr[0][i] != arr[2][i]) {
		isEqual3 = false;
	}
}
System.out.println("arr[0] == arr[1] ? : " + isEqual1); // false
System.out.println("arr[1] == arr[2] ? : " + isEqual2); // true
System.out.println("arr[0] == arr[2] ? : " + isEqual3); // true
```
Q. isEqual2의 값은 false여야하는데 true가 출력되는 이유?

A. if - else if 구조에서 if문이 true라면 else if는 실행하지 않고 넘어가기 때문
즉 if - else if는 전체 집합에서 if 조건을 검사한 뒤 false라면 전체 집합에서 if 조건 집합을 제외한 여집합에서 else if 조건을 검사하여 실행한다.
