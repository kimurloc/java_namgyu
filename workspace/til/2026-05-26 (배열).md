**Math.round()**
```java
double avg = 100/3.0;
avg = Math.round(avg); // 소수 첫째자리 반올림
avg = Math.round(avg*10)/10.0; // 소수 둘째자리 반올림
```
---
Q. `Math.round(avg*10)/10`의 결과 값이 `double`이 아닌 `int`인 이유?

A. `Math.round()`의 반환 타입은 `long`이기 때문에 `Math.round(avg*10)`연산에서 소수점이 모두 버려진다. 결과적으로 `long` / `int` == `int`이므로 소수 자리의 반올림을 위해선 `int`가 아닌 `double`이나 `float`타입을 나눠 주어야 한다.
### 9. 배열과 메모리 관리
#### 9.1 배열
같은 타입의 데이터 여러 개를 하나로 묶어서 순차적으로 관리하는 자료구조로, index(순서정보)를 이용하여 배열의 각 요소를 참조한다.
* 특징
	* **같은 타입만 저장 가능** : 하나의 배열에는 서로 다른 타입의 데이터를 저장할 수 없음
	* **고정된 크기** : 배열은 선언 및 생성할 때 크기를 지정해야 하며, 한 번 생성된 배열의 크기는 절대 변경할 수 없음
	* **연속된 메모리 공간** : 배열의 요소들은 메모리에 물리적으로 연속되게 배치되어 빠른 인덱스 접근이 가능함(배열의 크기를 변경할 수 없는 근거)
	* **참조 타입** : 배열 자체는 객체이므로 힙(Heap) 영역에 생성되며, 배열 변수는 이 객체의 주소값을 가짐

#### 9.2 배열 선언, 생성, 초기화
* 배열 선언
```java
type[] arrayName;
type arrayName[];
```
* 배열 생성
	실제 데이터를 저장할 공간을 메모리(`heap` 영역)에 할당하는 단계로, `new` 연산자와 함께 배열의 타입을 명시하고 크기를 지정
```java
arrayName = new type[size];
```
* 배열 생성 시 데이터 타입별 자동 초기화 기본값
	* `byte`, `short`, `int`, `long` : 0
	- `float`, `double` : 0.0
	- `char`: `'\u0000'` (공백 문자)
	- `boolean`: `false`
	- 참조 타입(`String`, 클래스 등) : `null`
- 배열 선언과 동시에 초기화
```java
type[] arrayName = {v1, v2, v3, ...};
type[] arrayName = new type[]{v1, v2, ...};
```

```java
Scanner s = new Scanner(System.in);
```

Q. 배열 선언 후 값을 할당하는데 왜 에러가 나는가?

A. 배열은 선언 후 초기화를 해주어야 비로소 생성됨!!!

#### 9.3 배열과 메모리 구조
- **스택(Stack) 영역**
    - 메서드 호출 시 생성되는 **로컬 변수**들이 적재되는 영역
    - **기본 타입 변수**는 **실제 데이터 값**을 스택에 가지고 있음
    - **참조 타입 변수**는 실제 객체가 저장되어 있는 **힙(Heap) 영역의 주소값**을 저장함
- **힙(Heap) 영역**
    - **`new` 연산자로 생성된 모든 인스턴스**와 **배열 객체**가 할당되는 영역
    - 가비지 컬렉터(GC)에 의해 참조 관계가 끊어진 객체들이 감지되어 자동으로 수거됨
