### 1. 프로그래밍의 이해
#### 1.1 컴파일러 vs 인터프리터
* **컴파일** : 개발자가 작성한 소스 코드를 컴퓨터가 이해할 수 있는 기계어로 한 번에 번역하는 과정
* **인터프리팅** : 소스 코드를 한 줄씩 읽어가며 기계어로 즉석에서 번역하는 과정
**컴파일러**는 컴파일을 수행하는 프로그램, **인터프리터**는 인터프리팅을 수행하고 그 결과를 즉시 실행하는 프로그램

#### 1.2 컴파일러 언어
* 코드를 실행하기 전, 전체 소스 코드를 한 번에 기계어로 번환(컴파일)하여 실행 파일을 만드는 방식
* 특징
	* 실행 속도는 매우 빠르지만, 코드를 수정할 때마다 모두 다시 컴파일해야 하므로 개발 속도는 상대적으로 느릴 수 있음
	* CPU 아키텍처마다 사용하는 명령어 집합이 다르기 때문에 그에 맞게 기계어가 생성되어야 하고, 프로그램이 메모리를 할당받거나 화면에 글자를 출력할 때 운영체제에 요청을 보내는 ``시스템 콜``이 운영체제마다 다름. 결과적으로 컴파일된 결과물은 CPU 아키텍처와 운영체제에 종속

#### 1.3 인터프리터 언어
* 코드를 실행할 때 전체를 미리 번역하지 않고, 한 줄씩 읽어가며 즉석에서 기계어로 번역과 실행을 동시에 수행하는 방식
* 특징
	* 실행 파일을 미리 만들지 않으므로 코드를 수정한 후 즉각적인 확인이 가능해 개발 속도가 빠름
	* 코드를 실행할 때마다 매번 번역을 거쳐야 하므로 컴파일러 언어에 비해 실행 속도는 상대적으로 느림
	* 각 운영체제에 맞는 인터프리터만 설치되어 있으면 동일한 코드를 어디서든 실행할 수 있어 플랫폼에 독립적

#### 1.4 하이브리드 언어
* 소스 코드를 `바이트코드(중간어)`로 1차 컴파일한 뒤, 실행 시점에 가상 머신을 통해 기계어로 번역하며 실행하는 혼합 방식
	> `바이트코드(bytecode , portable code, p-code)`는 **특정 하드웨어가 아닌 가상 컴퓨터에서 돌아가는 실행 프로그램을 위한 이진 표현법**이다. 하드웨어가 아닌 소프트웨어에 의해 처리되기 때문에, 보통 기계어보다 더 추상적이다.
* 특징
	* 컴파일 단계를 통해 실행 전에 문법 오류를 잡을 수 있고, 가상 머신이 실행 환경에 맞게 번역해주므로 운영체제에 독립적
	* 인터프리터의 느린 속도를 보완하기 위해 ``JIT 컴파일러`` 등을 활용하여 자주 쓰이는 코드를 기계어로 캐싱
		>`JIT(Just-In-Time) 컴파일러`는 **프로그램 실행 중(Runtime)에 자주 사용되는 코드(바이트코드)를 기계어로 실시간 번역하여, 인터프리터 방식의 느린 속도를 보완하고 성능을 극대화하는 기술**이다.

| 구분       | 컴파일러 언어                      | 인터프리터 언어                  | 하이브리드 언어                        |
| -------- | ---------------------------- | ------------------------- | ------------------------------- |
| 동작방식     | 실행 전 전체 소스 코드를 한 번에 번역함      | 실행 시점에 코드를 한 줄씩 즉석에서 번역함  | 1차 컴파일 후, 실행 시점에 가상 머신이 번역함     |
| 실행 속도    | 매우 빠름                        | 상대적으로 느림                  | 중간 (JIT 컴파일러로 보완함)              |
| 개발 속도    | 수정 시마다 재컴파일이 필요해 상대적으로 느림    | 실행 파일 생성 없이 즉시 확인이 가능해 빠름 | 컴파일 단계가 있으나, 가상 머신 덕분에 테스트가 용이함 |
| 하드웨어 독립성 | 종속적임 (CPU, OS별로 다르게 컴파일해야 함) | 독립적임 (인터프리터만 있으면 실행 가능함)  | 독립적임 (가상 머신 위에서 동일하게 동작함)       |
| 대표 언어    | C, C++, Rust, Go             | Python, JavaScript, Ruby  | Java, C#                        |
