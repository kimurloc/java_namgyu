### 10. 메서드

#### 10.1 메서드 선언
특정 기능을 수행하기 위해 관련된 코드들을 하나의 블록으로 묶어놓은 것
- 코드의 재사용성(Reusability) 증대(코드의 중복 제거)
- 유지보수성(Maintainability) 향상, 가독성(Readability) 및 구조화
```java
public int getSum(int n1, int n2) {
  int sum = n1 + n2;
  return sum;
}
```
- 접근제어자 : 메서드를 호출할 수 있는 범위를 지정
	- `public` : 어디서든 호출 가능
	- `private` : 같은 클래스 내부에서만 호출 가능
	- `protected` : 동일한 패키지 내의 클래스 / 상속 관계에 있는 자식 클래스에서 호출 가능
	- `default`(생략 시 기본 값) : 동일한 패키지 내의 클래스에서만 호출 가능
- 메서드를 선언할때 접근제어자, 반환타입을 명시하여야 한다.

#### 10.2 main 메서드
```java
public static void main(String[] args) {...}
```
- `public` : JVM이 어디서나 프로그램 시작점인 main 메서드에 접근하여 실행할 수 있어야 하므로 가장 넓은 범위인 public을 사용
- `static` : JVM이 클래스의 인스턴스 객체를 메모리에 생성하지 않고도 클래스 이름으로 바로 메서드를 호출하여 실행할 수 있도록 함
- `void` : 메서드의 실행이 끝나면 프로그램 자체가 종료되므로, 호출한 JVM에게 돌려줄 반환값(리턴값)이 없다는 것을 뜻함
- `String[] args` : 프로그램 실행 시 터미널 등 외부에서 전달하는 문자열 매개변수들을 배열 형태로 전달받음
**자바 21 이후 새로운 main 메서드**
- **암묵적으로 선언된 클래스 (Implicitly Declared Classes / Unnamed Classes)**
	- 클래스 선언('class 클래스명 { }')을 아예 생략하고 파일 내부에 바로 멤버 메서드만 작성하여 선언
	- 컴파일러가 소스코드 파일명을 클래스명으로 인식하여 내부적으로 암묵적인 클래스를 자동 생성함
	- 단, 외부 다른 클래스에서 사용할 수 없으므로 단독 실행용 클래스에서만 사용됨
	- 자바 IDE 프로젝트 탐색기에서는 완전한 자바 클래스 구조로 취급되지 않아 파일명 뒤에 '.java' 확장자가 그대로 표시되는 특징이 있음
- **인스턴스 메인 메서드 (Instance Main Methods)**
```java
void main() {...}
```
- 자바 21부터는 'static' 키워드와 'public' 키워드를 생략하고 'void main()' 형식으로 선언 가능
- 명령행 매개변수(`String[] args`)가 필요 없을 경우 생략 가능
- JVM이 클래스를 실행할 때 내부적으로 객체 인스턴스를 자동으로 먼저 생성한 후 이 main 메서드를 호출함
#### 10.3 메서드 오버로딩
