#### 5.3 상수
* 한 번 값이 저장되면 프로그램이 실행되는 동안 절대 변경할 수 없는 불변의 데이터 저장소
* 변수 선언부 앞에 `final` 키워드를 붙혀 상수 생성

#### 5.4 리터럴(Literal)과 식별자(Identifier)
* **리터럴** : 코드에서 프로그래머가 직접 입력한 고정된 값 자체, 변수나 상수에 할당되는 데이터 값
* **식별자** : 변수, 상수, 메서드, 클래스 등에 붙여주는 이름
`자료형` `식별자` = `리터럴`;

#### 5.5 형변환(Type Casting)
* 자동 형변환 (묵시적 형변환) : 작은 크기의 타입을 큰 크기의 타입으로 변환할 때 발생 (데이터 손실 없음)
* `byte` -> `short` -> `int` -> `long` -> `float` -> `double`
* 강제 형변환 (명시적 형변환) : 큰 크기의 타입을 작은 크기의 타입으로 강제 변환할 때 발생, 자료형을 명시해야 함(데이터 손실 위험)

### 6. 연산자
#### 6.1 연산자 우선순위
| 순위  | 연산자 유형         | 연산자 기호                | 연산 방향              |
| :-- | :------------- | :-------------------- | :----------------- |
| 1   | 괄호 및 대괄호       | (), []                | 왼쪽에서 오른쪽           |
| 2   | 단항 연산자         | ++, --, +, -, !       | 오른쪽에서 왼쪽           |
| 3   | 산술 연산자 (곱셈 계열) | *, /, %               | 왼쪽에서 오른쪽           |
| 4   | 산술 연산자 (덧셈 계열) | +, -                  | 왼쪽에서 오른쪽           |
| 5   | 비교 연산자 (크기 비교) | <, <=, >, >=          | 왼쪽에서 오른쪽           |
| 6   | 비교 연산자 (동등 비교) | ==, !=                | 왼쪽에서 오른쪽           |
| 7   | 논리 연산자 (AND)   | &&                    | 왼쪽에서 오른쪽           |
| 8   | 논리 연산자 (OR)    | \|                    | 왼쪽에서 오른쪽           |
| 9   | 삼항 연산자         | `조건`? `true`:`false`  | 오른쪽에서 왼쪽           |
| 10  | 대입 연산자         | =, +=, -=, *=, /=, %= | 오른쪽에서 왼쪽           |
| 11  | 증감 연산자(후위형)    | ++, --                | 2순위에 있지만 편의상 따로 표기 |

#### 6.2 연산 방향(결합성)의 의미
* 왼쪽에서 오른쪽 : 대부분의 산술, 비교, 논리 연산(일반적인 계산 순서)
* 오른쪽에서 왼쪽 : 대입 연산자, 단항 연산자에 해당. 우측의 식부터 해석하여 좌측으로 누적 및 대입해 들어옴
```java
a = b = c = 100; // 우측부터 해석하여 좌측으로 차례대로 저장해 들어옴
int y = -++x; // 우측인 ++x를 먼저 수행한 후, 그 결과에 좌측의 부호 연산자(-)를 씌움
```
