#### 11.8 캡슐화와 접근 제어자
- **캡슐화** : 중요한 데이터와 이를 처리하는 메서드를 하나로 묶고, 외부에서 임의로 변경하지 못하도록 데이터를 은닉하는 기법
- **접근 제어자** : ``private`` < ``default`` < ``protected`` < ``public`` 순서로 접근 권한을 제한
- 필드를 ``private``으로 선언하고 ``Getter/Setter`` 및 유효성 검증 메서드를 통해서만 접근하도록 제한

#### 11.9 접근 제어자별 접근 범위
| 접근 제어자    | 클래스 내부 | 동일 패키지 | 동일/자식 클래스 | 어디서나   |
| :-------- | :----- | :----- | :-------- | :----- |
| public    | 허용 (O) | 허용 (O) | 허용 (O)    | 허용 (O) |
| protected | 허용 (O) | 허용 (O) | 허용 (O)    | 차단 (X) |
| default   | 허용 (O) | 허용 (O) | 차단 (X)    | 차단 (X) |
| private   | 허용 (O) | 차단 (X) | 차단 (X)    | 차단 (X) |
- ``public`` : 어떠한 제한 없이 모든 곳에서 접근이 가능함
- ``protected`` : 동일한 패키지 내의 클래스 및 다른 패키지에 있더라도 해당 클래스를 상속받은 자식 클래스에서 접근이 가능함
- ``default (package-private)`` : 제어자를 생략했을 때 자동으로 부여되며, 동일 패키지 내부에서만 접근을 허용함
- ``private`` : 오직 선언된 해당 클래스 내부에서만 호출 및 접근이 가능함

#### 11.10 정적(static) 멤버
`static` 키워드가 붙은 필드나 메서드는 인스턴스 생성을 하지 않아도 클래스명으로 바로 접근 가능하며, 메서드 영역에 저장된다.
- `static`의 제한 사항
	- `static` 메서드는 객체 생성 전 클래스 로딩 시점에 메모리에 올라가므로, 메모리에 존재 여부가 불확실한 인스턴스 필드나 인스턴스 메서드에 직접 접근할 수 없음
	- `static` 메서드 내부에서는 오직 `static` 필드(클래스 변수)나 `static` 메서드만 직접 호출이 가능
	- 객체 자신을 가리키는 참조 키워드인 `this`는 `static` 메서드 내부에서 사용할 수 없음
```java
public class Calculator {
  int instanceVal = 10;      
  static int staticVal = 20; 

  public static void staticMethod() {
	// System.out.println(instanceVal); // error
    System.out.println(staticVal); // OK
  }

  public void instanceMethod() {
	System.out.println(instanceVal); // OK
    System.out.println(staticVal);   // OK
  }
}
```

#### 11.11 변수의 종류와 비교
| 구분     | 지역 변수                    | 인스턴스 변수 (Instance / field) | 클래스 변수<br>(Class / static) |
| :----- | :----------------------- | :------------------------- | :------------------------- |
| 선언 위치  | 메서드, 생성자, 블록 내부          | 클래스 블록 내부 (static 없음)      | 클래스 블록 내부 (static 있음)      |
| 저장 영역  | 스택 (Stack) 영역            | 힙 (Heap) 영역                | 메서드 (Method / Static) 영역   |
| 초기화 방식 | 자동 초기화 안 됨 (사용 전 초기화 필수) | 기본값으로 자동 초기화               | 기본값으로 자동 초기화               |
| 생성 시점  | 변수 선언문이 실행될 때            | 객체(인스턴스)가 생성될 때            | 클래스가 메모리에 로드될 때            |
| 소멸 시점  | 선언된 블록이 종료될 때            | 객체가 GC에 의해 소멸할 때           | 프로그램이 종료될 때                |
