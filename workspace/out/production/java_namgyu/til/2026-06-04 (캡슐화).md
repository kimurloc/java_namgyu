#### 11.8 캡슐화와 접근 제어자
- **캡슐화** : 중요한 데이터와 이를 처리하는 메서드를 하나로 묶고, 외부에서 임의로 변경하지 못하도록 데이터를 은닉하는 기법
- **접근 제어자** : ``private`` < ``default`` < ``protected`` < ``public`` 순서로 접근 권한을 제한
- 필드를 ``private``으로 선언하고 ``Getter/Setter`` 및 유효성 검증 메서드를 통해서만 접근하도록 제한

#### 11.9 접근 제어자별 접근 범위
| 접근 제어자    | 클래스 내부 | 동일 패키지 | 동일/자식 클래스 | 어디서나   |
| :-------- | :----- | :----- | :-------- | :----- |
| public    | 허용 (O) | 허용 (O) | 허용 (O)    | 허용 (O) |
| protected | 허용 (O) | 허용 (O) | 허용 (O)    | 차단 (X) |
| default   | 허용 (O) | 허용 (O) | 차단 (X)    | 차단 (X) |
| private   | 허용 (O) | 차단 (X) | 차단 (X)    | 차단 (X) |
- ``public`` : 어떠한 제한 없이 모든 곳에서 접근이 가능함
- ``protected`` : 동일한 패키지 내의 클래스 및 다른 패키지에 있더라도 해당 클래스를 상속받은 자식 클래스에서 접근이 가능함
- ``default (package-private)`` : 제어자를 생략했을 때 자동으로 부여되며, 동일 패키지 내부에서만 접근을 허용함
- ``private`` : 오직 선언된 해당 클래스 내부에서만 호출 및 접근이 가능함

#### 11.10 정적(static) 멤버
static 키워드가 붙은 필드나 메서드는 인스턴스 생성을 하지 않아도 클래스명으로 바로 접근 가능하며, 메서드 영역에 저장된다.
