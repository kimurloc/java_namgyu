### 12. 상속
#### 12.1 상속의 특징
상속은 **객체지향 프로그래밍의 핵심 개념** 중 하나로, 기존에 정의되어 있는 클래스의 모든 필드와 메서드를 새로운 클래스가 물려받아 사용하는 것을 의미함. 이때 상속을 해주는 기존 클래스를 **부모 클래스**(상위 클래스, `Super Class`)라 하고, 상속을 받는 새로운 클래스를 **자식 클래스**(하위 클래스, `Sub Class`)라고 함.
```java
public class Animal {
  public void eat() { System.out.println("eating"); }
}

public class Dog extends Animal {
  public void bark() { System.out.println("barking"); }
}
```
- **코드의 재사용성 향상** : 부모 클래스의 코드를 재사용하므로 중복 코드 감소
- **유지보수성 향상** : 부모 클래스의 수정만으로 이를 상속받은 모든 자식 클래스에 변경 사항이 반영되므로 일관성 있는 수정이 가능
- **다형성 구현의 기반** : 부모 클래스 타입의 참조 변수로 다양한 자식 클래스 객체를 참조할 수 있어, 확장성 높은 프로그램 설계가 가능
- **단일 상속 지원** : 자바는 다중 상속에서 발생할 수 있는 메서드 호출의 모호성(`다이아몬드 문제`)을 방지하기 위해 클래스의 단일 상속만 지원함. 즉, `extends` 뒤에는 단 하나의 클래스만 선언할 수 있음
	>`다이아몬드 문제`란 객체지향 프로그래밍에서 **다중 상속**할 때 발생하는 모호성 문제로, 공통 조상이 두 경로로 중복 상속될 때 어느 쪽을 따를 것인지 정하지 못하는 문제이다. 자바는 클래스 다중 상속을 금지시킴으로써 이러한 문제를 미연에 방지한다.

#### 12.2 메서드 오버라이딩(Overriding)
부모 클래스로부터 상속받은 메서드의 내부 구현을 자식 클래스에서 필요에 맞게 재정의하는 기법을 메서드 오버라이딩이라고 한다.
- 자식 클래스만의 기능 구현 : 부모 클래스의 일반적인 기능을 자식 클래스만의 구체적인 특성에 맞추어 재구성할 수 있음
- 다형성 구현의 핵심: 부모 클래스 타입의 참조 변수를 사용하더라도 실제 인스턴스의 오버라이딩된 메서드가 자동으로 호출되므로 다형성을 실현할 수 있음

#### 12.3 오버라이딩(Overriding) vs 오버로딩(Overloading)
| 구분     | 메서드 오버라이딩 (Overriding)     | 메서드 오버로딩 (Overloading)      |
| :----- | :------------------------- | :-------------------------- |
| 정의     | 상속받은 부모의 메서드를 자식 클래스에서 재정의 | 한 클래스 내에서 같은 이름의 메서드를 다중 정의 |
| 메서드 이름 | 부모 메서드와 동일함                | 동일함                         |
| 매개변수   | 동일함 (개수, 타입, 순서)           | 달라야 함 (개수 또는 타입, 순서)        |
| 반환 타입  | 동일해야 함                     | 상관없음                        |
| 접근 제어자 | 부모보다 좁아질 수 없음              | 상관없음                        |

#### 12.4 super와 super()
상속 관계에서 자식 객체 내부에는 부모 객체의 필드와 메서드가 함께 포함되므로, 이를 명확히 구분하고 생성자 흐름을 제어하기 위해 참조 키워드를 사용한다.
**super 참조 변수**
- 자식 클래스에서 부모 클래스의 필드나 메서드에 접근할 때 사용하는 참조 변수
- 멤버 변수 가려짐(Shadowing) 해결 : 자식 클래스에서 부모 클래스와 동일한 이름의 필드를 선언했을 때, 자식 클래스 내부에서 이름만으로 접근하면 자식 필드가 우선하므로 부모 필드에 접근하기 위해 `super.필드`을 명시적으로 사용함
**super() 생성자 호출**
- 부모 클래스의 생성자를 호출하는 함수
- 자식 클래스의 생성자가 호출되면 내부적으로 가장 먼저 부모 클래스의 생성자가 실행되어 부모 필드가 초기화된 후, 자식 생성자의 나머지 코드가 실행됨
- 첫 줄 배치 제약 : `super()`는 자식 클래스 생성자 코드의 첫 번째 줄에만 위치할 수 있음. 생략할 경우 컴파일러가 생성자 첫 줄에 자동으로 super()를 주입함
- 부모 클래스에 매개변수가 있는 생성자만 존재하고 기본 생성자가 정의되어 있지 않다면, 컴파일러가 자동 삽입하는 `super()`는 에러를 유발하므로 자식 생성자에서 직접 부모의 매개변수 생성자를 호출(`super(인수, ...)`)하도록 명시해야 함
