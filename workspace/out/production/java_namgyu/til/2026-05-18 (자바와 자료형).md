#### 1.5 절차지향 프로그래밍 (Procedural Programming)
* 프로그램의 실행 순서, 즉 절차를 중요하게 여기며 데이터를 처리하는 함수(명령의 집합) 중심으로 작성하는 방식
* 데이터보다 함수(동작)가 중심이 되며, 정해진 순서대로 처리하는 것에 집중
* 특징
	* 컴퓨터의 처리 구조와 유사하여 실행 속도가 빠름
	* 프로그램의 흐름을 파악하기 쉬움
	* 데이터와 함수가 분리되어 있어, 데이터 구조가 변경되면 관련 함수를 모두 수정해야 하는 번거로움이 있음
* 대표 언어 : C, Pascal, Fortran, COBOL

#### 1.6 객체지향 프로그래밍 (Object-Oriented Programming, OOP)
* 데이터와 그 데이터를 처리하는 함수를 `객체`라는 하나의 단위로 묶어 관리하는 방식
* 실제 세계의 사물을 모델링하여 설계
* 특징
	* 코드 재사용성이 높고 유지보수가 쉬움
	* 설계에 시간이 많이 소요되며, 절차지향에 비해 실행 속도가 상대적으로 느릴 수 있음
* 대표 언어 : Java, C++, Python, C#

### 2. 자바의 동작 방식
#### 2. 자바의 특징
* **운영체제 독립성** : JVM을 통해 어떤 플랫폼에서든 동일하게 실행 가능
* **순수 객체지향** : 설계부터 객체지향 원칙을 철저히 따름
* **자동 메모리 관리(GC)** : 개발자가 직접 메모리를 해제할 필요 없이 가비지 컬렉터가 찌꺼기 메모리를 자동 청소
* **멀티스레드 지원** : 언어 차원에서 강력한 멀티스레드 API를 제공

### 3. JVM의 동작 방식과 메모리
#### 3.1 JVM 코드 실행 순서
.java 파일 작성 -> 컴파일러(javac)가 바이트코드(.class)로 변환 -> JVM의 `클래스 로더`가 .class를 메모리로 적재 -> 실행 엔진이 한줄씩 읽으며 기계어로 변환 및 실행
> `클래스 로더`는 자바 런타임 환경(JRE)의 핵심 요소로, **실행 중에 자바 클래스 파일(.class)을 자바 가상 머신(JVM)으로 동적으로 읽어오는 역할**을 한다.

#### 3.2 실행 엔진 구성 요소
* **인터프리터** : 바이트코드(.class)를 한 줄씩 읽어서 네이티브 코드로 해석하고 즉시 실행
* **JIT (Just-In-Time) 컴파일러** : 인터프리터 방식으로 실행하다가, '핫 스팟(Hot Spot)'이라 불리는 자주 호출되는 코드 영역을 감지하면 해당 바이트코드 전체를 기계어로 컴파일하여 캐시에 저장하며, 이후에는 인터프리팅 없이 직접 실행됨
* **가비지 컬렉터** : 런타임 데이터 영역(Runtime Data Area)의 Heap 메모리를 관리하며, 더 이상 참조되지 않는 객체들을 식별하고 제거하여 메모리를 확보

#### 3.3 JVM 메모리 구조
* **메서드 영역 (Method Area / Static Area)**
	* 클래스 설계도 정보, 상수, `static` 변수가 저장되는 공유 공간
	* 프로그램 시작 시 할당되어 종료될 때까지 유지됨
* **스택 영역 (Stack Area)**
	* 메서드가 호출될 때마다 해당 메서드만의 작업 공간(스택 프레임)이 생성됨
	* 내부의 지역 변수, 매개변수가 저장됨
	* 메서드가 종료되면 메모리에서 즉시 해제 (LIFO 구조)
* **힙 영역 (Heap Area)**
	* `new` 키워드로 동적 생성되는 모든 객체(인스턴스)와 배열이 저장되는 거대한 공간
	* 스택 영역의 변수들은 이 힙 영역의 메모리 주소만을 가리킴(참조)
	* 어떤 변수에서도 참조하지 않는 객체는 가비지 컬렉터(GC)가 주기적으로 메모리에서 해제

### 5. 데이터 타입과 변수
#### 5.1 기본 데이터 타입(Primitive Type)
* 프로그래밍에서 더 이상 쪼갤 수 없는 가장 기본 단위의 데이터 타입
* **정수형**
	* `byte` (1 byte) : _-2^7 ~ 2^7-1 (-128 ~ 127)_
	* `short` (2 byte) : _-2^15 ~ 2^15-1 (-32,768 ~ 32,767)_
	* `int` (4 byte) : _-2^31 ~ 2^31-1_ (약 -21억 ~ 21억, 자바 정수 기본 타입)
	* `long` (8 byte) : _-2^63 ~ 2^63-1_ (약 -9.22E18 ~ 9.22E18, 끝에 `L` 또는 `l`을 붙임)
* **실수형**
	* `float` (4 byte) : 유효 숫자 7자리 정밀도 (정수부 포함, 끝에 `F` 또는 `f` 붙임)
	* `double` (8 byte) : 유효 숫자 15자리 정밀도 (정수부 포함, 자바 실수 기본 타입)
* **논리형과 문자형**
	* `boolean` (1 byte) : `true` 또는 `false`만 가짐
	* `char` (2 byte) : 홑따옴표로 문자 하나만 저장. 내부적으로는 유니코드(UTF-16) 숫자로 저장됨

#### 5.2 참조 데이터 타입(Reference Type)
기본 데이터 타입 8개를 제외한 모든 데이터 타입(배열, 클래스 등)은 참조 데이터 타입이고, 객체가 메모리 상에 실제 올라가 있는 위치 주소(Reference)를 가리키는 방식으로 작동한다.
