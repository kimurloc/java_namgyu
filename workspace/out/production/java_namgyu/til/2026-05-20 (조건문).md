### 7. 조건문
#### 7.1 새로 추가된 switch문(Switch Expression)
- 기존의 콜론(:)과 break 키워드를 사용하는 방식 대신 화살표 연산자(->)를 활용하여 가독성을 높이고 실수를 방지
- 기존 switch 문이 실행만을 목적으로 하는 제어문(Statement)이었다면, 새로운 switch 문은 그 자체로 최종 결과를 직접 반환할 수 있는 식(Expression)으로 동작 가능
* 예시(화살표 연산자)    
    ```java
    String role = "ADMIN";
    switch (role) {
        case "ADMIN" -> System.out.println("관리자 권한");
        case "USER" -> System.out.println("일반 사용자");
        default -> System.out.println("권한 없음");
    }
    ```
- 예시(다중 조건 결합):    
    ```java
    String day = "SATURDAY";
    String type = switch (day) {
        case "MONDAY", "TUESDAY", "WEDNESDAY", "THURSDAY", "FRIDAY" -> "평일";
        case "SATURDAY", "SUNDAY" -> "주말";
        default -> "알 수 없음";
    };
    System.out.println(type); // '주말' 출력
    ```
- 예시(switch 식 대입)    
    ```java
    int number = 2;
    String result = switch (number) {
        case 1 -> "하나";
        case 2 -> "둘";
        default -> "기타";
    };
    System.out.println(result); // '둘' 출력
    ```
- 예시(블록과 yield 키워드 사용):    
    ```java
    int score = 90;
    String grade = switch (score) {
        case 100, 90 -> {
            System.out.println("우수한 성적입니다.");
            yield "A"; // 중괄호 블록 내의 최종 반환값 지정
        }
        case 80 -> "B";
        default -> "C";
    };
    System.out.println("최종 학점: " + grade); // '최종 학점: A' 출력
    ```

Q. `price *= 1 - discountRate;`를 계산할 때, 분명 우변에 `double`인 `discountRate`가 섞여 있어서 `double` 연산이 일어나는데도 `price`가 `int`로 유지되는 이유?

A. Java에서 `*=`, `+=`, `-=` 같은 복합 대입 연산자는 **강제 형변환(Explicit Cast)** 코드가 자동으로 추가된다.

`price *= 1 - discountRate;` 코드는 내부적으로 단순히 `price = price * (1 - discountRate);`로 바뀌는 게 아니라
```java
price = (int) (price * (1 - discountRate));
```
의 과정을 거쳐 `price`가 `int`자료형을 유지하도록 만든다.
만약 복합 대입 연산자를 쓰지않고, `price = price * (1 - discountRate);`를 그대로 쓰게 된다면, 좌변은 `int`형, 우변은 `double`형 변수이므로 에러가 발생한다.
